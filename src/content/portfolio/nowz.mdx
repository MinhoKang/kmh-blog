## 프로젝트 개요

기존 **Recoil 기반으로 구축된 학생 관리 프로그램**의 상태 관리 시스템을 **Tanstack-Query로 전면 이전**하고, Chart.js를 활용한 **학생 분석 레포트 시스템**을 새롭게 구축한 프로젝트다.

대시보드, 수업 관리, 과제 관리, 학생 분석 레포트, 일정 관리 등 **종합적인 학생 관리 기능**을 제공하는 시스템이다.

## 핵심 기능

### 1. 상태 관리 시스템 개선

**기존 Recoil로 관리되던 전역 상태**를 Tanstack-Query 기반으로 전면 재구축했다.

**개선 내용**:

- 서버 상태와 클라이언트 상태의 명확한 분리
- 자동 캐싱 및 동기화 메커니즘 도입
- API 호출 최적화 및 중복 요청 제거
- 에러 처리 및 로딩 상태 표준화

### 2. 학생 분석 레포트 시스템

**Chart.js를 활용**하여 학생들의 학습 데이터를 시각화하는 분석 레포트 페이지를 개발했다.

| 차트 유형     | 분석 내용           | 활용 목적             |
| ------------- | ------------------- | --------------------- |
| **선형 차트** | 학습 성과 추이 분석 | 시간별 성장 패턴 파악 |
| **막대 차트** | 과목별 성취도 비교  | 강점/약점 과목 식별   |
| **원형 차트** | 과제 완료율 분석    | 학습 참여도 측정      |
| **히트맵**    | 출석률 시각화       | 학습 패턴 분석        |

### 3. 종합 학생 관리 기능

**대시보드 중심**의 통합 관리 시스템을 구축했다.

**주요 기능**:

- **대시보드**: 핵심 지표 및 알림 통합 관리
- **수업 관리**: 수업 일정 및 커리큘럼 관리
- **과제 관리**: 과제 생성, 제출, 평가 시스템
- **일정 관리**: 개인별/그룹별 일정 관리
- **학생 분석**: 성과 분석 및 레포트 생성

## 기술적 도전과 해결

### 1. Recoil에서 Tanstack-Query로의 상태 관리 이전

**문제**: 기존 Recoil 기반 전역 상태 관리의 한계점

**상황 분석**:

- 서버 상태와 클라이언트 상태가 혼재되어 관리 복잡성 증가
- API 캐싱 및 동기화 로직의 수동 구현 필요
- 네트워크 상태에 따른 에러 처리 일관성 부족

**해결책**:

| 해결 방법                | 구현 내용                      | 효과                  |
| ------------------------ | ------------------------------ | --------------------- |
| **서버 상태 분리**       | Tanstack-Query로 API 상태 관리 | 자동 캐싱 및 동기화   |
| **클라이언트 상태 유지** | Recoil로 UI 상태 관리          | 기존 로직 호환성 유지 |
| **에러 처리 표준화**     | Query Error Boundary 구현      | 일관된 에러 처리      |
| **로딩 상태 최적화**     | Suspense와 함께 사용           | 사용자 경험 향상      |

### 2. Chart.js 데이터 시각화 구현

**문제**: 학생 데이터의 효과적인 시각화 필요

**해결책**:

| 차트 유형          | 데이터 소스          | 시각화 효과    |
| ------------------ | -------------------- | -------------- |
| **성과 추이 차트** | 과제 점수, 시험 성적 | 학습 진도 파악 |
| **출석률 히트맵**  | 출석 데이터          | 학습 패턴 분석 |
| **과목별 비교**    | 과목별 성취도        | 강약점 식별    |

## 결론

NOWZ 학생 관리 프로그램의 상태 관리 개선 프로젝트를 통해 **기존 시스템의 점진적 개선**과 **새로운 기술 스택 도입**의 균형을 경험했다.

**Recoil에서 Tanstack-Query로의 전면 이전**을 통해 서버 상태와 클라이언트 상태를 명확히 분리하고, **Chart.js를 활용한 데이터 시각화**로 사용자에게 더 나은 분석 도구를 제공했다.

이 프로젝트는 **기존 코드베이스를 유지하면서도 새로운 기능을 안정적으로 추가**하는 **점진적 개선의 중요성**을 보여주는 사례다.

앞으로도 **사용자 경험을 개선**하고 **기술적 부채를 줄이는** 방향으로 지속적인 시스템 개선을 진행하고자 한다.
