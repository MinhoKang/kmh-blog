## 프로젝트 개요

나우즈 케어(원격) 고객인 **학부모와 학생이 사용하는 모바일 앱**을 React Native + Expo로 개발하여 **앱스토어와 구글 플레이에 출시**.  
실시간 채팅과 학생의 하루를 보고하는 일일 관찰 페이지가 핵심 기능.

---

## 기술 스택

- 프레임워크: React Native + Expo(SDK 53) + React 19
- 언어: TypeScript
- 상태 관리: tanstack-query, zustand
- 스타일링: nativewind, clsx
- 통신: socket.io, axios
- 폼 유효성 검사: react-hook-form + zod
- 모니터링/에러: sentry
- 개발 워크플로우: husky

### 3. 컴포넌트 개발

1. 개발 기능
   - @gorhom/bottom-sheet 라이브러리 활용하여 bottm-sheet 컴포넌트 및 ui 컴포넌트 개발

2. 개발 기간
   - 2025년 2월 ~ 2025년 3월

3. 해결한 문제

<ProblemBox
  title="문제 1: 확장 가능한 컴포넌트 API 설계"
  situation="향후 바텀시트 기능 확장과 다양한 사용 케이스 대응이 빈번할 것으로 예상되는 비즈니스 요구사항. 서드파티 라이브러리(@gorhom/bottom-sheet)의 복잡한 API를 직접 노출할 경우, 개발자마다 다른 방식으로 사용하게 되어 일관성 부족 및 유지보수 어려움 발생."
  solution="useImperativeHandle를 활용하여 컴포넌트 API 설계 → 일관된 API 인터페이스 제공, 라이브러리 업데이트 대응력 확보, 확장 가능한 컴포넌트 구조 구현"
/>

### 4. 토큰 관리

1. 개발 기능
   - axios 인터셉터를 통해 JWT 토큰 자동 갱신 시스템 구축
   - 토큰 만료시 갱신 후 원본 요청 재시도
   - 토큰이 갱신 중이라면 이후 들어오는 요청들은 대기 상태로 처리(갱신 요청 하지 않음)
   - SecureStore를 통해 토큰 저장 및 관리

2. 개발 기간
   - 2025년 3월

3. 해결한 문제

<ProblemBox
  title="문제 1: 토큰 갱신 시 중복 요청 방지"
  situation="토큰 갱신 시 중복 요청이 발생하여 서버 부하 증가 및 토큰 갱신 실패 가능성 증가"
  solution="전역 Promise isRefreshing 플래그 변수를 활용하여 중복 요청 방지"
/>

<ProblemBox
  title="문제 1: 사용자 업데이트 알림 부족"
  situation="EAS-update로 업데이트 할 경우 재심사가 필요 없는 장점이 있지만 사용자에게 업데이트 여부를 알려주지 않아 사용자가 업데이트를 모르고 있는 문제"

/>
